# Cursor 高级用法 - MCP 课程大纲
目标用户：混合受众（零基础 + 有基础的开发者）
目标字数：1500字 PPT 文案

---

## 一、引入：为什么需要 MCP？（痛点切入）
- 现有 AI 助手的局限：不知道你的数据库有什么、看不到文件系统、无法调用 API
- 每次都要手动复制粘贴信息，效率低下
- MCP 是解决方案：让 AI 能主动"看到"外部世界

## 二、MCP 是什么？（概念建立）
- 全称：Model Context Protocol，Anthropic 推出的开放协议
- 一句话理解：给 AI 装了一个「USB-C 接口」，统一标准连接任意外部工具
- MCP vs 传统 AI 助手对比（表格）：外部工具访问、实时数据获取、系统操作、自定义工作流
- 核心价值：AI 从「被动建议者」→「主动执行者」

## 三、MCP 工作原理（机制理解）
- 三层架构：MCP Host（Cursor）→ MCP Client → MCP Server → 外部服务
- 完整调用链：用户输入 → AI 解析意图 → 选择工具 → Server 执行 → 返回结果
- 重点：整个过程对用户透明，AI 自动决定何时调用工具
- 两种传输方式：stdio（本地，推荐新手）vs SSE/HTTP（远程，适合团队）

## 四、快速上手：三步配置 MCP（入门实操）
- 前置要求：Cursor 0.45.6+、Node.js 18+
- 配置文件位置：`~/.cursor/mcp.json`（全局）/ `.cursor/mcp.json`（项目级）
- 配置文件基本结构（command / args / env）
- 第一个实战：配置 Fetch Server（无需 Token，最简单）
- 验证方式：让 AI 抓取一个网页内容

## 五、进阶应用：高价值 MCP 场景（场景驱动）
- 场景 1：GitHub 集成 → 创建 PR、管理 Issues、查看分支（需 Token）
- 场景 2：数据库直连 → AI 自动分析表结构、生成精确 SQL（MySQL/PostgreSQL）
- 场景 3：文件系统操作 → 扫描项目、批量清理未用 import
- 场景 4：网页内容抓取 → 给 AI 一个链接直接分析文章
- 推荐入门 Server 清单（4个常用）

## 六、最佳实践与安全须知（进阶提升）
- 配置策略：全局配置放通用 Server，项目配置放专有 Server
- 安全三原则：mcp.json 加入 .gitignore / 用环境变量传敏感信息 / 最小权限
- 高效使用技巧：给 AI 明确指令比模糊提示效果好 10 倍
- 常见报错排查：JSON 格式错误 / Token 权限不足 / 重启 Cursor 生效

## 七、总结与展望
- 回顾核心：MCP = AI 的能力扩展标准接口
- MCP 生态趋势：一次编写，到处使用；生态快速增长
- 推荐下一步：从 Fetch Server 开始，逐步添加需要的 Server

# Cursor MCP：给你的 AI 装上「工具箱」— 逐字稿（零基础版）

---

hello 大家好，欢迎来到我的 Cursor 课程。上节课我们把 Cursor 的基础操作都熟悉了一遍，这节课我们来讲一个非常非常实用的功能——MCP。

先不着急解释 MCP 是什么，我想先问问大家，在用 Cursor 的时候，有没有遇到过这种情况：你让 AI 帮你做个事，比如说「帮我总结一下这篇文章」，然后你就得把文章全文复制，粘贴到对话框里，再让它帮你总结。或者你问 AI「帮我看看我电脑上这个文件」，它告诉你「我无法访问你的本地文件」。

这种感觉是不是有点像，你雇了个很厉害的助理，但他坐在一个没有窗户的房间里，只能靠你把信息一条一条口头传递给他，效率是非常的低。

这节课我们学的 MCP，就是帮这个助理打开窗户、装上电话、配上工具的方法。学完之后，你的 AI 就能自己去查资料、自己读文件、自己帮你完成很多原来需要你手动操作的事情。

我们这节课会讲以下几个部分：第一，MCP 到底是什么，用大家都能懂的方式来解释；第二，它能帮我们做哪些具体的事；第三，最重要的——手把手带大家配置第一个 MCP，五分钟就能搞定；第四，教大家怎么跟 AI 说话，才能让它真正帮上忙；最后是遇到问题怎么办。好，我们开始。

---

## 一、AI 的「局限」，你感受到了吗？

在正式讲 MCP 之前，我想先帮大家把这个问题搞清楚——为什么 AI 那么聪明，有些事情却做不到？

其实原因很简单。Cursor 里面的 AI，它默认只有一种能力，就是「聊天」。你给它文字，它给你文字。它没有办法自己去打开网页，没有办法读取你电脑里的文件，也没有办法去调用各种软件。

我们可以做一个形象的比喻。AI 就像一个坐在玻璃房间里的超级顾问，他脑袋里装了海量的知识，但他被困在里面出不来。你想让他帮你查资料，只能隔着玻璃把资料递给他；你想让他帮你操作电脑，他够不到。

MCP 做的事情，就是给这个玻璃房间开了一个口子，装上了各种传递工具——网络、文件、各种系统——让 AI 能够伸出手来，自己去拿需要的东西。

---

## 二、MCP 是什么？用三句话讲清楚

好，概念上来了。MCP 的全称是 Model Context Protocol，这个名字比较技术化，大家不用记住，记住这三句话就够了。

第一句：**MCP 是一套规则**，定义了 AI 应该怎么跟外部工具沟通。

第二句：**MCP 是一个工具箱**，装好之后 AI 就能用里面的工具——可以读网页、可以查文件、可以连数据库。

第三句：**MCP 用起来完全透明**，你不需要告诉 AI「去用那个工具」，你只需要正常说你想要什么，AI 自己知道该怎么办。

我再给大家一个类比，就非常好理解了。大家知道 USB-C 接口吗？以前手机充电，iPhone 是一种接口，安卓是另一种，笔记本电脑又是第三种，充电线买了一堆还经常找不到对的。后来出了 USB-C，一种接口通吃所有设备。

MCP 就是 AI 世界里的 USB-C 接口。不管是读网页的工具、管理文件的工具，还是连接数据库的工具，都用同一套规则来跟 AI 对接。你装了哪个工具，AI 就能用哪个工具，非常的统一，非常的方便。

---

## 三、装上 MCP，AI 能帮你做什么？

光说概念还不够直观，我们来看几个真实的场景，大家感受一下这个变化有多大。

**场景一：读网页**。这是最适合零基础同学的场景，今天我们也主要学这个。以前你想让 AI 帮你总结一篇文章，你需要全文复制粘贴给它。装上了 Fetch MCP 之后，你只需要说「帮我打开这个链接，总结一下文章的主要内容」，AI 自己就去抓取了，完全不需要你手动复制任何东西。那种感觉挺神奇的，第一次用的时候我自己也觉得很惊喜。

**场景二：管理 GitHub**。如果你有 GitHub 账号，这个就非常有用。配置了 GitHub MCP 之后，你可以直接跟 AI 说「查一下我的仓库里有没有标记为 bug 的未解决问题」，AI 自己去 GitHub 上查，把结果汇总给你，不用你手动点进去一条条看。

**场景三：整理文件**。如果你的项目文件夹乱七八糟，可以让 AI 帮你扫描，找出没用的文件，或者整理文件结构。这个功能稍微复杂一点，我们今天不深入讲，大家知道有这个可能性就好。

好，三个场景里面，今天我们重点来做第一个，读网页，因为它完全不需要任何编程知识，五分钟就能搞定，是最适合零基础同学入手的第一步。

## 四、动手之前：两个快速检查

好，我们进入实操环节。开始配置之前，需要做两个小检查。

**第一个检查：Cursor 版本。** 打开 Cursor，点击顶部菜单里的 Help，找到 About，看一下版本号。版本号需要是 0.45.6 或者更高。如果版本太低，在 Help 菜单里找 Check for Updates，点一下更新就好了，这个很简单。

**第二个检查：Node.js 有没有安装。** 这个可能稍微陌生一点，Node.js 是让 MCP 工具能跑起来的一个运行环境，简单来说就是一个必要的「地基」。检查方法也很简单，打开 Cursor 里面的终端——快捷键是 Ctrl 加反引号（键盘上数字 1 左边那个键）——然后输入 `node -v` 回车。如果看到一个版本号，比如 v20 什么的，说明已经装好了；如果显示「命令未找到」，就去 nodejs.org 下载，选 LTS 版本，点安装就行了，跟装普通软件一样，不需要任何额外设置。

## 五、手把手配置：Fetch MCP

好，两个检查都没问题了，我们来配置第一个 MCP——Fetch Server。

先解释一下，Fetch Server 是一个专门用来抓取网页内容的工具。装上之后，AI 就能帮你读任何网页，而且完全不需要账号、不需要密钥，是所有 MCP 里面门槛最低的一个，非常适合入门。

**第一步，打开配置界面。** Windows 或者 Linux 系统，按 Ctrl+Shift+J；macOS 系统，按 Cmd+Shift+J。打开之后，在左侧找到 Tools & Integrations，然后点最下面的 New MCP Servers，这时候会打开一个文本文件，这就是我们的配置文件。

**第二步，把代码复制进去。** 把下面这段内容，原封不动地复制到那个文本文件里：

```json
{
  "mcpServers": {
    "fetch": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    }
  }
}
```

这段代码看起来有点奇怪，大家不用理解里面每个字的意思，照着抄就行了。唯一要注意的是，大括号、引号、冒号都不能少，缺一个都会出错。如果你不确定有没有复制对，可以把它贴到 jsonlint.com 这个网站上检查一下格式。

**第三步，保存文件。** Ctrl+S 或者 Cmd+S，保存。

**第四步，也是最关键的一步——重启 Cursor。** 这一步大家一定要记住，不是刷新页面，不是切换窗口，是彻底关掉 Cursor，然后重新打开。我见过非常多的同学配置完了说「我的 MCP 不管用」，结果发现只是忘了重启，重启一下就好了。

重启之后，我们来验证一下。打开对话框，说这句话：「帮我打开 https://cursor.com 这个网站，告诉我这个工具是做什么用的」。如果 AI 回复了网站上的内容，那就说明配置成功了！

## 六、学会说「对的话」，效果差 10 倍

MCP 装好了，但好不好用，还有一半取决于你怎么说话。

我举个例子。同样是想让 AI 帮你总结文章，你说「帮我查一下」——AI 不知道查什么，去哪查；你说「帮我读这篇文章」——AI 可能读了但不知道要输出什么格式给你。

效果好的说法是什么呢？要包含三个要素：**要什么、从哪拿、用什么格式给我**。

比如：「帮我打开这个链接 [URL]，总结文章的 3 个核心要点，用列表格式展示」。这三个要素都说清楚了，AI 一次就能给你想要的结果，不需要来回反复确认。

我给大家准备了几个可以直接用的模板，大家收藏一下。

总结文章用这个：「帮我打开 [链接]，用 5 句话总结这篇文章的主要内容」。

提取关键词用这个：「帮我读取 [链接]，提取文章里提到的所有工具名称，整理成列表」。

对比两篇文章用这个：「帮我分别打开这两个链接，对比一下它们的主要区别，列出 3 点不同」。

这几个模板可以直接套用，里面把链接换成你实际的链接就行了。

## 七、遇到问题，别慌

最后说说几个常见的报错情况，帮大家少走弯路。

**情况一：配置好了，AI 还是说无法访问。** 第一件事，检查有没有重启 Cursor。真的，这是最常见的原因，80% 的情况重启一下就解决了。

**情况二：保存配置文件的时候报错。** 这通常是代码格式有问题，多了一个引号或者少了一个大括号。把配置代码复制到 jsonlint.com 检查一下格式，它会告诉你哪里有问题。

**情况三：`node -v` 显示命令未找到。** 去 nodejs.org 下载 LTS 版本，安装完之后，重启一下电脑，再试一次。

**情况四：完全不知道哪里出了问题。** 这个也很好办，把你看到的错误信息截图，然后直接问 Cursor：「我在配置 MCP 的时候遇到了这个错误，请帮我分析原因和解决方法」。AI 擅长处理这类问题，它会帮你找出原因。

## 总结

好，我们来总结一下今天学的内容。

MCP 是给 AI 装上工具箱的一套规则，让 AI 从「只会聊天」变成「能主动帮你干活」。我们今天配置的 Fetch Server，是入门门槛最低、最容易上手的一个，装上之后 AI 就能帮你读任何网页，再也不用手动复制粘贴了。

记住那个口诀：跟 AI 说话要说清楚三件事——要什么、从哪拿、用什么格式给你。这样 AI 一次就能给你想要的结果。

如果你用顺了 Fetch MCP，下一步可以探索 GitHub MCP，让 AI 帮你管理代码仓库；或者文件管理 MCP，让 AI 直接操作你的项目文件夹。Cursor 官方还有一个 MCP 目录，里面有很多可以一键安装的工具，大家可以去逛逛，地址是 cursor.com/cn/docs/context/mcp/directory。

记住：从最简单的开始，先把一个用顺了，再慢慢加新工具。工具用的多不如用的熟，这个道理在这里同样适用。

下节课，我们将进入 Cursor Rules 功能——如何让 AI 记住你的偏好，每次生成的结果都更符合你的风格。敬请期待！

## 开始之前：2 个检查项

**检查 1：Cursor 版本**
- 打开 Cursor，点击菜单 `Help` → `About`
- 版本号需要 ≥ **0.45.6**
- 版本太低？点击 `Help` → `Check for Updates` 更新即可

**检查 2：Node.js 是否安装**
- 打开 Cursor 里的终端（快捷键：`` Ctrl+` ``）
- 输入 `node -v`，回车
- 看到版本号（如 `v20.0.0`）说明已安装 ✅
- 看到「找不到命令」→ 去 nodejs.org 下载安装，选 **LTS 版本**

---

## 打开 MCP 配置界面

**第一步：打开设置**

| 系统 | 快捷键 |
|------|--------|
| Windows / Linux | `Ctrl + Shift + J` |
| macOS | `Cmd + Shift + J` |

**第二步：找到 MCP 配置**  
在左侧菜单找到 `Tools & Integrations`  
点击底部的 `New MCP Servers` 按钮

**第三步：会弹出一个文本文件**  
这就是你的 MCP 配置文件，接下来把代码复制进去就行。

---

## 复制这段代码（照着抄，不需要理解）

把下面这段完整复制进配置文件：

```json
{
  "mcpServers": {
    "fetch": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    }
  }
}
```

**注意事项：**
- 如果文件里已经有内容了，不要全部删掉，找专业人士帮忙合并
- 确保大括号、引号都完整，缺一个字符都会出错
- 保存文件（`Ctrl+S` 或 `Cmd+S`）

---

## 最关键的一步：重启 Cursor！

> ⚠️ **90% 的人配置失败，都是因为忘了这一步！**

保存配置文件之后，**必须完全关闭 Cursor，然后重新打开**。

不是点刷新，不是切换窗口——是**彻底关掉再开**。

重启之后，打开对话框，说这句话测试：

> 「帮我打开 https://cursor.com 这个网站，告诉我这个工具是做什么用的」

如果 AI 回复了网站内容 → 🎉 配置成功！  
如果 AI 说「我无法访问外部链接」→ 检查代码格式是否有误

## 学会说「对的话」：让 AI 100% 听懂你的意图

MCP 装好了，但效果好不好，还取决于你怎么说。

**对比示例：**

| 说法 | 效果 |
|------|------|
| ❌「帮我查一下」 | AI 不知道查什么、去哪查 |
| ✅「帮我打开这个链接 [URL]，总结文章里的 3 个主要观点，用列表格式展示」 | AI 精准完成任务 |

**零基础口诀（记住这三要素）：**

```
① 要什么（总结 / 翻译 / 提取）
② 从哪拿（链接 / 文件路径）  
③ 怎么给我（列表 / 表格 / 一段话）
```

---

## 练习一下：3 个可以直接用的提示词模板

**模板 1：总结文章**
> 「帮我打开 [链接]，用 5 句话总结这篇文章的主要内容」

**模板 2：提取要点**
> 「帮我读取 [链接]，提取文章里提到的所有工具名称，整理成列表」

**模板 3：对比分析**
> 「帮我分别打开这两个链接：[链接1] [链接2]，对比它们的主要区别」

把这几个模板收藏起来，随时可以用。

## 遇到问题别慌：4 种常见情况

**情况 1：AI 还是说「无法访问」**  
→ 检查有没有重启 Cursor（重启是第一步！）

**情况 2：配置文件保存时报错**  
→ 把配置代码全部复制，去 jsonlint.com 粘贴检查格式

**情况 3：`node -v` 显示「命令未找到」**  
→ 去 nodejs.org 下载 LTS 版本，安装后重启电脑再试

**情况 4：不知道哪里出了问题**  
→ 把错误信息截图，直接问 Cursor：「我在配置 MCP 时遇到了这个错误，怎么解决？」

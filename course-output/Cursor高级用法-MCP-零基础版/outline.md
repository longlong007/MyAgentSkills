# Cursor 高级用法 - MCP 课程大纲（零基础版）
目标用户：零基础小白（无编程背景，不了解 API/数据库等概念）
目标字数：1500字 PPT 文案

## 设计原则
- 聚焦「是什么、能做什么、怎么装」，完全回避底层原理和技术术语
- 每个概念必须有生活化类比
- 实操以「最简单、零门槛」的 Fetch Server 为核心案例
- 绝不出现：JSON-RPC、stdio/SSE、架构、协议栈等技术词汇

---

## 一、引入：你遇到过这个问题吗？（痛点共鸣）
- 用 Cursor 提问时，AI 说「我没有权限访问你的文件/数据库」
- 要把大量信息手动复制粘贴给 AI，效率很低
- AI 很聪明，但它是个「瞎子」——看不到你电脑里的东西
- 今天的主角 MCP，就是给 AI 装上「眼睛和手」

## 二、MCP 是什么？（用生活类比讲透）
- 全名不重要，关键理解：MCP = AI 的「万能遥控器」
- 类比 1：手机遥控家电 → MCP 让 AI 遥控你电脑里的各种工具
- 类比 2：USB-C 接口统一标准 → 一套规则，连接所有工具
- 一句话总结：MCP 让 AI 从「只会聊天」变成「能干活」

## 三、MCP 能帮你做什么？（以结果驱动，场景化展示）
- 场景 1：给 AI 一个网址，让它帮你总结文章内容（不用复制粘贴）
- 场景 2：让 AI 帮你在 GitHub 上创建分支、查看问题（需要知道 GitHub 的人）
- 场景 3：让 AI 自动整理项目文件夹（删除多余文件）
- 重点聚焦：场景 1 最适合零基础，今天主要学这个

## 四、零门槛上手：5 分钟配置第一个 MCP（Fetch Server）
- 前置检查：Cursor 版本够不够（怎么看）+ Node.js 怎么装（一句话）
- 打开配置界面（截图式步骤说明）
- 复制粘贴这段代码进去（完整代码，不需要理解，照着抄就行）
- 保存 + 重启 Cursor
- 验证成功：说一句话测试

## 五、学会用 MCP：怎么跟 AI 说话才有效？
- 错误示范：「帮我查一下」→ AI 不知道查什么、去哪查
- 正确示范：「帮我打开这个链接，总结文章的 3 个核心要点」
- 小白口诀：「说清楚你要什么 + 从哪里拿 + 用什么格式给我」
- 动手练习提示词示例

## 六、一定要知道的注意事项（安全 + 避坑）
- 配置完之后要重启 Cursor（最常见的问题）
- 有些 MCP 需要「钥匙」（Token），不要随便把钥匙给别人
- 配置文件不要上传到网上
- 遇到报错怎么办：复制错误信息问 Cursor「这是什么意思」

## 七、总结：你学会了什么？
- MCP = 给 AI 装上了工具箱，让它能主动帮你干活
- 今天学会了最简单的 Fetch MCP，可以让 AI 帮你读网页
- 下一步：探索更多 MCP（GitHub、文件管理）

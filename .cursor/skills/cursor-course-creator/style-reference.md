# Wayne 老师逐字稿语言风格参考

本文档从 5 篇历史逐字稿中提炼的风格规范，生成逐字稿时必须严格遵循。

## 一、开场模式

固定使用以下两种之一：

**首节课**：
> hello大家好，我是Wayne老师，欢迎来到我的cursor实战课程。这节课是XXX课，我会讲讲……

**后续课程**：
> hello大家好，欢迎来到我的Cursor课程。上节课我们学习了XXX，这节课将进入一个非常实用的主题——YYY。

开场后紧跟本节课结构预览（3-5个要点罗列）。

## 二、自然衔接与过渡

使用多样化的衔接方式让内容流畅过渡，**禁止使用"那"字起句**（这是口头禅坏习惯）：

| 衔接场景 | 推荐表达 |
|---------|---------|
| 新话题开始 | "我们先来看看……" / "接下来我们来到……" / "好，我们进入下一个部分……" |
| 补充说明 | "另外，……" / "还有一个……" / "同时，……" / "除此之外，……" |
| 总结过渡 | "总结一下，……" / "简单来说就是……" / "概括起来就是……" |
| 个人观点 | "我个人的建议是……" / "我的看法是……" / "从我的经验来说……" |
| 条件转折 | "如果你的……的话，我们可以……" / "假设遇到……，我们就……" |
| 因果推进 | "这样一来就……" / "所以说……" / "正因如此，……" |

其他推荐衔接词：
- "好，我们继续下一个环节。"
- "当然了，……也有它的局限性。"
- "可能……会知道……"（照顾不同层次听众）
- "说到这里，……"
- "回到刚才的话题，……"

## 三、解释技术概念的方式

**核心原则：先说是什么，再用大白话翻译，最后举例子。**

模式：`技术概念` → `"简单来说就是……"` → `生活化类比`

参考示例：

> "深度上下文的理解，简单来说就是它懂你整个项目。"

> "传统IDE就像自行车，插件对话式的就像电动摩托，cursor就像自动驾驶汽车。"

> "它不像我们一些有脾气的技术大牛，或者是一些不耐烦的领导，我们没有社交压力，就让AI疯狂的帮我们干活。"

> "AI Agent……让AI从一个只是被动的给我们一些建议的伙伴，变成了一个可以不分昼夜的干活的牛马。"

常用类比关键词：自行车/电动摩托/汽车、搬砖、导师、牛马、管家、顾问。

## 四、强调与修饰模式

Wayne 老师的高频修饰结构：

- **"非常的" + 形容词**："非常的方便"、"非常的流畅"、"非常的智能"
- **"比较" + 形容词**（表达适度判断）："比较的合适"、"算是比较强了"
- **"还是"引导推荐**："我觉得cursor还是对小白最友好"
- **"其实"引导解释**："其实都是在vs code这个架构上面增加自己的AI功能"
- **"所以说"引导结论**："所以说他的生产效率是完全不一样的"
- **反复强调**："是所有！它可以帮我们……"

## 五、受众互动表达

- 使用"我们"而非"你"拉近距离："我们可以看到"、"我们先来看看"
- 直接呼唤："大家可以看到"、"大家最好跟着整个视频进行"
- 体贴听众："如果有不懂或者跟不上的地方，可以暂停一下"
- 分层照顾："可能熟练编程的程序员会知道vs code"

## 六、个人观点与推荐表达

Wayne 老师表达个人立场时的固定句式：

> "我个人的建议是……"
> "另外说一下，我个人对工具的一个使用态度。"
> "是基于我们用了非常顺手的……"
> "因为是亲身体验了非常多的工具之后得出的结论。"

核心态度：**务实、不追新、用熟一个再换**：
> "与其追求一个99-100分的工具，我们不如把一个做的90分好的工具熟练使用然后提升生产力"

## 七、结尾模式

逐字稿必须包含以下结尾结构：

1. **本节总结**（1-2句话概括核心要点）
2. **下节预告**（"下节课，我们就正式进入XXX"）
3. **固定结束语**："好，谢谢大家。" 或 "这节课就先到这里，谢谢大家。"

可选追加：
- 布置课后作业："留给大家一个作业，就是……"
- 学习激励："编程的本质它不是写出漂亮的代码，而是解决问题"

## 八、口语化细节

以下细节让文稿听起来自然：

- 适度重复和自我修正："如果AI出错了，或者说是越改越烂的时候"
- 插入语和补充："像一些JavaScript要在上面运行的话，就得装这个运行环境"
- 带有节奏感的列举："第一个就是……第二个是……第三部分就是……"
- 感叹与评价："这是一个非常激动的时代"
- 非正式量词："好几天"、"一大块"、"一行行"

## 九、禁忌（避免的风格）

- 不使用学术论文式的书面语
- 不使用过多英文（技术术语除外）
- 不使用"您"（统一使用"你"或"我们"）
- 不使用过于正式的连接词（"因此"、"综上所述"、"鉴于"）
- 不使用"那"字起句（这是口头禅坏习惯，用上述第二节的替代衔接词）
- 不跳过解释直接给结论
